# ThreadsPosts

## Summary

Threads アカウント運用を「ナレッジ収集 → 投稿生成 → Threads Graph API（`graph.threads.net/v1.0`）直接投稿 → 計測 → フィードバック」まで一気通貫で回す自動化パイプライン。リポジトリは [tatoflam/ThreadsPosts](https://github.com/tatoflam/ThreadsPosts)（Private）、Node.js + YAML frontmatter + OpenSpec ワークフローで運用。初代アカウント [@chokatsu_studio](https://www.threads.net/@chokatsu_studio)（腸活・表示名: 超活スタジオ🌱）は 2026-04-29 に本番初投稿（post_id `17978831033843102` = D011）に成功したが、2026-05-04 に Meta の自動分類器で BAN。以後 interior ジャンルの account_1 = oheyamemo（お部屋メモ）へ pivot し、Local-First アーキテクチャ + warmup プロトコルの下で楽天アフィリエイト収益化（月 30 万 → 3 アカ段階 scaling で ¥900k）を目指している。

### 画像

- L1 = 撮影 / L2 = Unsplash（`/image-pick-web`: draft → 英語 query → 候補 → user 選択 → `dept/content/assets/<account>/<draft_id>/` + frontmatter `image_paths:` `image_license:`、プレビューは `dept/content/assets/account_1/D00X/index.md`）/ L3 = 楽天実商品（未実装、当面 user 手動目視・保存）/ L4 = AI 生成 mood
- L4: `npm run image:gen -- <draft-id>`（preview）→ `--adopt`（assets コピー + frontmatter 更新）= `/image-gen <draft-id>`。provider 既定 Google Gemini 画像（"Nano Banana"、`GEMINI_API_KEY`）。gen cache `~/.config/threads-posts/image_library/gen_cache/<draft-id>/`（TTL 30 分）。採用時 frontmatter `image_generation:`（provider / model / prompt / seed）
- 外部ホストは Google Drive 案採用（`image-hosting-gdrive` change、Cloudflare R2 / S3 / Cloudinary 比較の末「GWS アカウントがあるから簡潔な実装で」）

> [!note] Gemini 画像は課金必須 (無料枠 0)
> セットアップで 429 が出た時の原因切り分け: `GEMINI_API_KEY` 有効でも **画像生成だけ
> 429** なら Google 側 billing 未設定を疑う ([[reference_gemini_image_billing.md|memory: gemini-image-billing]] でメモリ化済)。
> Gemini text API は無料枠が残るが、画像 API は別カテゴリ。

> [!info] visual_verified は WebFetch の clean 判定を信用しない
> 過去 wltech で WebFetch がランキングバナーを誤判定した前例 = 「目視検証必須」が
> プロジェクト方針。実商品画像の clean/non-clean 判定は user 目視 + frontmatter
> `visual_verified` 注記が canonical で、自動判定は補助情報扱い。
