4. `/weekly-report-sync-baseline`（前週 Doc snapshot を carry-forward 差分元に。導入前世代は 1 回だけ skip）

7. Milestone 再取得（pm-master-backup 再実行）
8. 工程グリッド再生成（writeback を反映）
9. `/weekly-report-publish run_date=YYYY-MM-DD` — Doc 冪等 in-place update + HTML dist + Firebase Hosting deploy（URL 不変なので再配信・修正版上書きが安全）

### Firebase 認証切れリカバリ（publish の予期される失敗モード）

- 症状: publish 最終ステップで `Authentication Error: Your credentials are no longer valid. Please run firebase login --reauth`（`FIREBASE_TOKEN` 未設定 → 保存済み login token 失効。Step 0 通過後でも 1〜2h で失効しうる）
- 回復: user が手動で `firebase login --reauth`（対話 OAuth、非対話代替なし）→ **publish 部分のみ再実行**: `cd plugins/drive-publisher && PYTHONPATH=scripts python -m publish --run-date YYYY-MM-DD`。Drive アップロード成功済みなら Doc は冪等 update で重複作成なし、`latest_state.json` の doc ポインタ更新まで完走させる
