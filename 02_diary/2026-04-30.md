  2. **`rewrite-seed-for-variant-keyed-workbook` 提案＋実装**（`96b20e5` / `6bd9be6` / `b75afdf`）— 顧客が `部材リスト_20260430.xlsx` を新列構造で再支給。旧 layout は「D 列以降が flat な option 列」だったが、**新 layout は D/E/F 列 = Natural/Flat/Sharp の per-variant default、G+ 列 = alternative options**。`scripts/extract-finish-options.mjs` を全面書き換え:
     - ヘッダー行を読んで Natural / Flat / Sharp 列を**列名で動的に発見**（D/E/F 固定の前提を撤廃）
     - 各 part header 行で「白/白/黒」のような重複を**1 option per distinct label に collapse**して `defaultForVariants: VariantKey[]` を埋める（"白" → `[natural, flat]`、"黒" → `[sharp]`）。3 option / 2 重複だった旧出力が消える
     - 製品コードは header 行**直下の sub-row（row + 1）**から、サブラベル（②の「電球色」「光無し」等）も同サブ行から取得
     - 画像 anchor は `xl/drawings/drawing*.xml` を読んで、ラベルとは**異なる column × row の swatch cell** にマッチさせる（新 workbook はヘッダー直下の行に画像を置く）
     - `SPECDRAWING_WORKBOOK` env-var fallback を撤去、`部材リスト.xlsx` を canonical 名に戻す（`old/` 配下に旧版を歴史保存）。`finishOptionSchema` に `defaultForVariants: z.array(z.string()).default([])` 追加。`b75afdf` で `OVERVIEW_JA.md` / spec / design / tasks / `AUTHORING.md` を新 layout 準拠に更新
  3. **収束フェーズ：cache-bust + collapsed-label texture override + ⑮ 床 を texture へ flip**（`5836a21` / `5d1dcbb` / `6c9fcac`）— 顧客が部材リスト xlsx に追記したアイコン / 製品コードが画面に反映されない問題を 3 commit で解決:
