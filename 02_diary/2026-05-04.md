### 学び（横展開可能）

- **「コスト判断を人手に残す」opt-in 設計**: デフォルト無料、`--asr` で明示承認しないと有料 API は走らせない。月額予算の超過リスクは運用人手判断で吸収できる。「自動 fallback で気づかぬ間に課金爆発」より圧倒的に安全
- **LLM の system prompt は出力例を ` ``` ` で囲むと literal コピーされる**: Claude が ` ``` ` 込みで markdown を返してきた件は、system prompt の例示にコードフェンスを使ったのが直接原因。出力例は plaintext で、プログラム側でフォーマットを固定するのが安全
- **frontmatter を LLM に任せない**: model 名 hallucination や `(unknown)` メタデータの問題は、frontmatter を Claude が生成すると不可避。プログラム側で確実に上書きする pattern が安定運用には必須
- **多言語パスのテストは「コミュニティ翻訳字幕」で簡単に迂回される**: TED-Ed が JA 字幕を持っていたために `youtube-transcript` の言語優先順位 `ja > en` が真の EN→JA 翻訳パスをスキップした。多言語 smoke は「英語のみで JA 字幕がない動画」を選ぶ必要がある（BBC など報道系が確実）
- **「ローカル Node v25 + openai v4 SDK + ESM」の組合せで multipart upload が ECONNRESET**: openai SDK v4 → v6 upgrade で解消。Node v25 環境では SDK のメジャーバージョンを最新で揃える運用に
- **Whisper の音声誤字は Claude が文脈で吸収する**: ASR の生 transcript には固有名詞の誤字が含まれるが（「イイコリ」→ `E. coli`、「肝菌」→ `グラム陰性桿菌`）、Topics 生成段階で Claude が文脈から正しく解釈する。「ASR の精度が低くても下流の LLM が補正する」のは ASR + LLM 二層構成の堅牢性の根拠
- **OpenSpec の `## ADDED Requirements` wrapper は spec sync 時に削除する**: change-side の delta は wrapper 形式だが、main spec に sync するときは wrapper を外して plain spec に変換する。Purpose セクションは change-side からはコピーされないので別途追加が必要
