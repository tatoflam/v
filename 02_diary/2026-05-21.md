### IAM 権限の罠 — `developer1@` の Service Usage Admin だけでは `iamcredentials.googleapis.com` を enable できない

- `firebase deploy` が auto-enable する API（Eventarc 等）のため、`developer1@` を **Service Usage Admin** に格上げ → ここまでは通る
- だが Gmail API + DWD のために `gcloud services enable iamcredentials.googleapis.com --project=todobot-dev` を `developer1@` で叩くと **`PERMISSION_DENIED` (`AUTH_PERMISSION_DENIED`)**
- 原因: org policy 側で `iamcredentials` 系の enable と SA 自己 Token-Creator binding は **super-admin 限定**
- 解決: `--account=thomma@meguru-construction.com` で明示
  ```bash
  gcloud services enable iamcredentials.googleapis.com \
    --project=todobot-dev --account=thomma@meguru-construction.com
  gcloud iam service-accounts add-iam-policy-binding \
    1085648053016-compute@developer.gserviceaccount.com \
    --member='serviceAccount:1085648053016-compute@developer.gserviceaccount.com' \
    --role='roles/iam.serviceAccountTokenCreator' \
    --project=todobot-dev --account=thomma@meguru-construction.com
  ```
- → 他プロジェクト（habi-bff など）で **`iamcredentials` 系の enable に詰まったら同じ手順を流用**できる reusable pattern
