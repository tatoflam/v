# Tasks — 道路使用許可 窓口申請 効率化ツール

## 1. 環境セットアップ（GWS）

- [ ] 1.1 組織アカウントで管理表スプレッドシートを作成し、`sites` / `slots` / `config` / `notifications` の4シートと列ヘッダ・データ検証・凡例（参考表示の注記）を整備する
- [ ] 1.2 Drive にルート共有フォルダを作成し、秘書課・バックオフィス・現場監督の共有権限を設定する
- [ ] 1.3 スタンドアロン GAS プロジェクトを作成し、管理表・ルートフォルダの ID を `config` に登録する
- [ ] 1.4 `doc03.docx` を Google Document に変換し、期間欄へ `{{PERIOD_FROM}}` / `{{PERIOD_TO}}` プレースホルダを設定したテンプレート Doc を作成する（提出日欄は空欄・バーコードは無変更）
- [ ] 1.5 【ユーザー確認①】変換後テンプレートの体裁（罫線・バーコード・様式再現度）をクライアントと確認し、必要なら Docs 上で様式を修正する

## 2. 管理表・スロット展開（permit-schedule-management）

- [ ] 2.1 `sites` の読み取り・バリデーション（必須項目チェック、案件ID採番）を実装する
- [ ] 2.2 申請期間を14日刻みで分割し `slots` へ展開する処理を実装する（終端切り詰め・回次採番）
- [ ] 2.3 再実行時の冪等性（既存スロット保持、期間延長時の末尾追加のみ）を実装する
- [ ] 2.4 カスタムメニュー（「スロット展開」「申請書生成」「巡回実行」）を管理表に追加する

## 3. フォルダ整備・資料巡回（drive-folder-monitoring）

- [ ] 3.1 案件登録時の `現場名/工種/` + サブフォルダ3種の自動作成（既存再利用）と、工種フォルダ URL の `sites` への書き戻しを実装する
- [ ] 3.2 図面セット（3ファイル）・受理済写しの格納状況を判定し `slots` へステータスを書き戻す巡回処理を実装する
- [ ] 3.3 巡回処理を日次の時間主導型トリガーとして登録する

## 4. 申請書生成（application-doc-generation）

- [ ] 4.1 テンプレート複製 → プレースホルダ置換（期間の和暦/西暦・時刻フォーマット）→ `01_申請書` への配置 → Doc URL 書き戻しを実装する
- [ ] 4.2 生成済みスロットのスキップ（再生成防止）と命名規則を実装する
- [ ] 4.3 置換漏れ検査（`{{` が残っていたらエラー表示）を実装する

## 5. アラート通知（alert-notification）

- [ ] 5.1 営業日計算（土日 + 日本の祝日カレンダー）ユーティリティを実装する
- [ ] 5.2 受領日未入力スロットの判定と GmailApp によるアラート送信（現場名・工種・期間・警察署・管理表リンクを本文に含む）を実装する
- [ ] 5.3 通知ログによる重複送信防止（1日1回）と、開始日超過時の件名エスカレーションを実装する
- [ ] 5.4 送信失敗時の管理者通知とログ記録を実装する
- [ ] 5.5 アラート処理を毎営業日朝の時間主導型トリガーとして登録する

## 6. 検証・受入・本番展開

- [ ] 6.1 テスト案件（ダミー現場×工種）で「登録 → 展開 → 生成 → 巡回 → アラート」の一連の動作を検証する
- [ ] 6.2 【ユーザー確認②】進行中の実案件を登録し、生成された申請書をクライアント（現場監督・秘書課）と確認する。期間欄の時刻表記を警察署の受理実績に合わせて確定する
- [ ] 6.3 アラート宛先を管理者のみにして1週間試運転し、誤報・漏れを調整する
- [ ] 6.4 【ユーザー確認③】試運転結果をレビューし、通知先を本番宛先（秘書課・担当監督）へ切り替えて本番運用を開始する
- [ ] 6.5 運用手順書（案件登録手順、LINE 報告の転記手順、期間短縮時の手動アーカイブ手順）を作成し共有する
