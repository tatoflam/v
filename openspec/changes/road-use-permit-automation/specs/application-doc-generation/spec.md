# application-doc-generation — 申請書 Google Document 自動生成

## ADDED Requirements

### Requirement: スロットごとに申請書ドキュメントを自動生成する
システムはスロット展開後、各スロットについてテンプレート Google Document（別記様式第六・道路使用許可申請書）を複製し、案件の工種フォルダ内 `01_申請書` サブフォルダへ保存しなければならない（SHALL）。生成した Doc の URL は `slots` シートの該当行に書き戻さなければならない（SHALL）。

#### Scenario: 全スロット分の一括生成
- **WHEN** 3スロットを持つ案件に対してドキュメント生成処理を実行する
- **THEN** `01_申請書` フォルダに3件の申請書 Doc が「道路使用許可申請書_{現場名}_{工種}_{第n回}_{開始日}-{終了日}」の命名で作成され、各スロット行に URL が記録される

#### Scenario: 生成済みスロットのスキップ
- **WHEN** 申請書 Doc URL が既に記録されているスロットに対して生成処理を再実行する
- **THEN** 当該スロットの Doc は再生成・上書きされない

### Requirement: 期間欄へスプレッドシートの値を転記する
生成される申請書の「期間」欄（テンプレートの黄色ハイライト部）には、スロットの開始日時・終了日時を「YYYY年M月D日H時 から YYYY年M月D日H時 まで」の形式で転記しなければならない（SHALL）。

#### Scenario: 期間プレースホルダの置換
- **WHEN** スロット 8/1 0時〜8/14 24時 の申請書を生成する
- **THEN** 期間欄に開始・終了の年月日時が転記され、プレースホルダ文字列（`{{...}}`）は文書中に残らない

### Requirement: 提出日欄とバーコードは変更しない
生成される申請書の冒頭の提出日欄（年月日）は空欄のままとし（MUST）、テンプレートに含まれるバーコードは変更せずそのまま複製しなければならない（SHALL）。

#### Scenario: 手書き運用のための空欄維持
- **WHEN** 申請書 Doc が生成される
- **THEN** 冒頭の「年　月　日」欄は空欄であり、窓口提出時にペンで手書きできる状態である

#### Scenario: バーコードの保持
- **WHEN** 申請書 Doc が生成される
- **THEN** テンプレートのバーコード画像が位置・内容とも変更されずに含まれている
