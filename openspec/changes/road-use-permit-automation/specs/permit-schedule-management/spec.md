# permit-schedule-management — 管理表と2週間スロット管理

## ADDED Requirements

### Requirement: 案件マスタで現場×工種を管理できる
管理表スプレッドシートの `sites` シートは、1行を「現場×工種」として、案件ID・現場名・工種・申請期間開始日・申請期間終了日・担当課・管轄警察署・責任者・通知先メールアドレスを保持しなければならない（SHALL）。

#### Scenario: 案件の新規登録
- **WHEN** 秘書課が `sites` シートに現場名・工種・申請期間・担当課・管轄警察署・通知先を入力し登録処理を実行する
- **THEN** システムは一意の案件IDを採番し、当該行を有効な案件として扱う

#### Scenario: 必須項目の欠落
- **WHEN** 申請期間開始日または終了日が未入力の行に対して展開処理が実行される
- **THEN** システムはその行をスキップし、ステータス列にエラー理由を表示する

### Requirement: 申請期間を2週間スロットへ自動分割する
システムは案件の申請期間 [開始日, 終了日] を、開始日から14日刻みのスロット列に分割し、`slots` シートへ1スロット1行で展開しなければならない（SHALL）。最終スロットの終了日は申請期間終了日を超えてはならない（MUST NOT）。

#### Scenario: 6週間の申請期間の展開
- **WHEN** 申請期間が 8/1〜9/11（6週間）の案件に対して展開処理を実行する
- **THEN** `slots` シートに 8/1〜8/14、8/15〜8/28、8/29〜9/11 の3スロットが回次付きで作成される

#### Scenario: 14日で割り切れない期間
- **WHEN** 申請期間が 8/1〜8/20 の案件に対して展開処理を実行する
- **THEN** 8/1〜8/14 と 8/15〜8/20（終端で切り詰め）の2スロットが作成される

#### Scenario: 再実行時の冪等性
- **WHEN** 既にスロット展開済みの案件に対して展開処理を再実行する
- **THEN** 既存スロット行は変更・重複作成されず、期間延長で新たに必要になったスロットのみ末尾に追加される

### Requirement: 受領・次回申請のステータスを記録できる
`slots` シートは、スロットごとに受領日（道路使用許可証の窓口受領日）と次回申請日（次スロットの申請提出日）の手入力欄を持たなければならない（SHALL）。

#### Scenario: 受領報告の転記
- **WHEN** 現場監督から LINE で受領報告を受けたバックオフィスが当該スロット行の受領日を入力する
- **THEN** 当該スロットのアラート対象判定は解消され、以後アラートは送信されない
