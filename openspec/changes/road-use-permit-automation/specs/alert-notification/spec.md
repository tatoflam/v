# alert-notification — 申請漏れ防止メールアラート

## ADDED Requirements

### Requirement: 受領日未入力スロットへ始期日前アラートを送信する
システムは毎営業日の定時に全スロットを走査し、「スロット開始日の N 営業日前（N は設定値、既定3）を過ぎている」かつ「受領日が未入力」のスロットについて、案件の通知先メールアドレスへ GmailApp でアラートメールを送信しなければならない（SHALL）。メール本文には現場名・工種・スロット期間・管轄警察署・管理表へのリンクを含めなければならない（SHALL）。

#### Scenario: 始期日3営業日前の未受領アラート
- **WHEN** スロット開始日が3営業日後で、受領日が未入力のまま日次アラート処理が実行される
- **THEN** 通知先メールへ現場名・工種・期間・管轄警察署・管理表リンクを含むアラートが送信される

#### Scenario: 受領済みスロットは対象外
- **WHEN** 受領日が入力済みのスロットに対して日次アラート処理が実行される
- **THEN** アラートは送信されない

#### Scenario: 土日祝の除外
- **WHEN** アラート判定の先行日数を計算する
- **THEN** 土日および日本の祝日は営業日に数えない

### Requirement: 同一アラートの重複送信を防止する
システムは送信したアラートを通知ログ（送信日時・スロットID・種別・宛先）に記録し、同一スロット・同一種別のアラートは1日1回を超えて送信してはならない（MUST NOT）。

#### Scenario: 同日の再実行
- **WHEN** 同じ日にアラート処理が2回実行される
- **THEN** 既に当日送信済みのスロットへは再送信されない

#### Scenario: 未解消アラートの翌営業日再送
- **WHEN** アラート送信の翌営業日になっても受領日が未入力のまま処理が実行される
- **THEN** アラートが再度送信され、スロット開始日を過ぎている場合は件名で超過が強調される

### Requirement: 送信失敗時に管理者へ通知する
アラートメールの送信に失敗した場合、システムはエラー内容を管理者メールアドレスへ通知し、通知ログに失敗として記録しなければならない（SHALL）。

#### Scenario: 無効な宛先
- **WHEN** 案件の通知先メールアドレスが無効で送信が失敗する
- **THEN** 管理者へエラー通知が送られ、通知ログに失敗が記録される
