# drive-folder-monitoring — Drive フォルダ整備と資料ステータス巡回

## ADDED Requirements

### Requirement: 現場・工種のフォルダ階層を自動作成する
システムは案件登録時に、ルートフォルダ配下へ `現場名/工種/` のフォルダ階層と、その中に `01_申請書` `02_図面セット` `03_受理済写し` のサブフォルダを作成しなければならない（SHALL）。作成済みの場合は再作成せず既存フォルダを使わなければならない（SHALL）。工種フォルダの URL は `sites` シートに書き戻さなければならない（SHALL）。

#### Scenario: 新規案件のフォルダ作成
- **WHEN** 新しい現場×工種の案件が登録され初期化処理が実行される
- **THEN** `現場名/工種/` と3つのサブフォルダが作成され、工種フォルダの URL が `sites` シートに記録される

#### Scenario: 同一現場の別工種追加
- **WHEN** 既存の現場に別工種の案件が追加される
- **THEN** 現場フォルダは再利用され、工種フォルダのみ新規作成される

### Requirement: 図面セットと受理済写しの格納状況を定期巡回する
システムは時間主導型トリガーで毎営業日1回以上、各案件の `02_図面セット` フォルダ（案内図・保安図・迂回路図）と `03_受理済写し` フォルダの格納状況を確認し、`slots` シートの該当列にステータス（充足/不足）を書き戻さなければならない（SHALL）。

#### Scenario: 図面セット充足の反映
- **WHEN** 現場監督が `02_図面セット` に案内図・保安図・迂回路図の3ファイルを格納した後、巡回処理が実行される
- **THEN** 当該案件のスロット行の図面セット状況が「充足」表示に更新される

#### Scenario: 前回写しの不足検知
- **WHEN** 第2回スロットの申請時期が近づいているが `03_受理済写し` に第1回の受理済み申請書写しが格納されていない状態で巡回処理が実行される
- **THEN** 当該スロット行の前回写し状況が「不足」表示に更新される

### Requirement: 巡回ステータスは参考情報である
格納状況の判定はファイル数・命名に基づく簡易判定とし、ファイル内容の妥当性は保証しない。管理表上でその旨を明示しなければならない（SHALL）。

#### Scenario: 判定基準の明示
- **WHEN** 利用者が管理表のステータス列を参照する
- **THEN** 列ヘッダまたは凡例に「ファイル格納の有無による参考表示」であることが明記されている
