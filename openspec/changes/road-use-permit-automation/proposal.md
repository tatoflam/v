# 道路使用許可 窓口申請 効率化ツール（GAS）

## Why

道路使用許可のオンライン申請（eGov）が警察側の処理停滞により機能せず、窓口申請へ切り替えることになった。道路使用許可は **2週間分ずつしか取得できない** ため、工事期間中は「受領 → 次回申請」のループを現場×工種ごとに回し続ける必要があり、手作業では申請漏れ・書類作成の重複作業が発生するリスクが高い。株式会社めぐるの Google Workspace（GWS）組織アカウント上で、軽量な GAS ベースの仕組みにより申請書生成・ステータス管理・アラートを自動化する。

## What Changes

- **管理表（Google Spreadsheet）の新設**: 現場名／工種（材料搬入・残土搬出・生コン打設 等）／申請期間／担当課／管轄警察署／2週間ごとの期間スロット／受領日・次回申請日の入力欄を持つマスタ兼ステータス管理表を定義する。
- **申請書の自動生成**: 工種ごとに申請期間を管理表に指定すると、GAS が期間を2週間単位に分割し、全スロット分の申請書を Google Document として工種フォルダ内に自動生成する。申請書の様式は警察様式「別記様式第六・道路使用許可申請書」（`doc03.docx` 相当）に準拠し、黄色ハイライトの「期間」欄（開始年月日時〜終了年月日時）にスプレッドシートの値を転記する。冒頭の提出日は手書き運用のため空欄のまま、バーコードは固定で変更しない。
- **Drive フォルダの自動整備と資料ステータス巡回**: 「現場 > 工種」の階層で Drive フォルダを自動作成し、図面セット（案内図・保安図・迂回路図）や受理済み申請書写しの格納状況を GAS の定期トリガーで巡回し、管理表にステータス反映する。
- **メールアラート**: 各2週間スロットの始期日の数日前（既定: 3営業日前、設定可能）になっても受領日が入力されていない場合、担当者へ GmailApp でアラートメールを送信する。
- **入力運用**: 受領・次回申請の完了報告は現場監督から LINE で受け、秘書課またはバックオフィスが管理表へ日付を手入力する（LINE 連携の自動化は本変更のスコープ外）。

## Capabilities

### New Capabilities
- `permit-schedule-management`: 管理表（Spreadsheet）のデータモデルと、申請期間を2週間スロットへ分割・展開する機能。受領日／次回申請日のステータス管理を含む。
- `application-doc-generation`: 管理表の指定に基づき、警察様式に準拠した申請書 Google Document を全スロット分、工種フォルダへ自動生成する機能。
- `drive-folder-monitoring`: 「現場 > 工種」フォルダ構成の自動作成と、図面セット・受理済み申請書写しの格納状況の定期巡回・ステータス反映。
- `alert-notification`: 受領日未入力スロットの始期日前アラートメール送信と、送信履歴の管理（重複送信防止）。

### Modified Capabilities

（なし — 新規システムのため既存 spec への変更はない）

## Impact

- **新規リソース**: GWS 組織アカウント配下に GAS プロジェクト1本、管理表 Spreadsheet 1冊、申請書テンプレート Google Document 1件、Drive 共有フォルダツリー（現場 > 工種）を新設。
- **利用サービス**: SpreadsheetApp / DocumentApp / DriveApp / GmailApp / 時間主導型トリガー（すべて GWS 標準・追加費用なし）。
- **運用への影響**: 現場監督は図面セットを Drive に格納するのみ。秘書課・バックオフィスは LINE 報告を受けて管理表に日付を入力するのみ。申請書の Word 手作業コピー・日付書き換え作業が不要になる。
- **スコープ外**: LINE Messaging API 連携、eGov 再連携、手数料支払い管理、印刷そのものの自動化。
