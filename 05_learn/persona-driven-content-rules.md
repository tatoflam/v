# ペルソナ駆動の継続可能なコンテンツルール

## Summary

健康・サプリ系の Threads アカウント [@chokatsu_studio](https://www.threads.net/@chokatsu_studio)（[[03_work/threadsposts]]）で、本人が **D011 アカウント始動以来の第 2 投稿で詰まった**経験から導いた継続可能性のためのルール。「煽り感」「権威借用に見える」「確信が得られず続けられない」という詰まりは、技術ではなく**ペルソナ設計と文体ルール**で解消できる。

## 詰まりの正体

書き手の側が以下の 3 点で確信を持てなくなると、第 2 投稿が打てない:

1. **煽り感** — 「〜していないか？」「〜は危険だ」のような問い詰め型・恐怖訴求が「健康インフルエンサーの定型」と同化する違和感
2. **権威借用に見える** — PIVOT・Dr.石黒・教授の発言を引用するとき、書き手が「権威に乗っかっている」ように見えるのではないかという疑念
3. **確信が得られない** — 出典つき事実を並べるほど、自分自身の体験や判断との距離感がぼやける

## 解決の中核 — ペルソナを「実践者・キュレーター」に固定

書き手が自身を**「医師でも販売者でもない、いち実践者・キュレーター」**として定義する。

- **役割**: 専門家の発信を読み込んで自分なりに咀嚼し、「自分はこう受け取った」を共有する
- **一人称**: **「私」**（「皆さん」「あなたたち」のような上から目線の主語は使わない）
- **読者との距離**: 同じ目線で考えている人。教えに行かない、煽らない
- **出典の扱い**: 権威付けではなく**出発点**として使う。本文中で肩書を強調せず、末尾 1 行に脚注として添える

このペルソナ定義は **CLAUDE.md / SKILL.md / posting_policy.md** に書き込み、AI が文体生成するときの基底に据える。

## 投稿の 5 ルール

1. **1 投稿 500 文字以内**
2. **冒頭 1 行で結論を一人称で言い切る**（例: 「私は〜を疑うようになった」「私は〜だと考えている」）
3. **具体的な数字や固有名詞を必ず入れる**
4. **出典は本文中で権威付けに使わない。末尾 1 行に脚注として添える**
5. **最後の 1 行は命令ではなく、読者が自分で検証できる問い・観察ポイントにする**

## 禁止表現と言い換え

| 禁止表現 | 理由 | 言い換え |
|---|---|---|
| 「〜かもしれません」 | 曖昧だと信頼されない | 一人称で言い切る（「私はこう見ている」） |
| 「〜と言われています」 | 出所のない伝聞は弱い | 誰が言ったか明記 |
| 「〜していないか？」「〜していい？」 | 問い詰め型は煽り感 | 観察ポイントで締める |
| 「〜は危険だ」「逆効果になる」単独 | 恐怖訴求のみで終わると健康インフルエンサーの定型と同化 | 体験/解釈で挟む |
| 「変える時だ」「今すぐ〜しよう」 | 上から目線の命令 | 問いで締める |

## 文体テンプレ — 4 段構成（事実+体験/解釈ハイブリッド）

健康ジャンルでは **「出典つき事実 + 自分の解釈/体験」のハイブリッド**が信頼と人格を両立させる。純粋数字＋出典なしは煽り垢、純粋体験は N=1 で弱い、**挟み込み型が最強**。

1. **結論断言**（1 行目、一人称で言い切る）
2. **出典明記つき事実**（誰が・どこで言ったか — 「医師・石黒成治氏の YouTube 解説より」「PIVOT『がんを防ぐ腸活』で〜」のように **人物名 + プラットフォーム/番組名** を主張の前に挿入）
3. **自分なりの解釈/体験**（書き手の咀嚼）
4. **観察ポイントとしての問い**（読者が自分で検証できる）

frontmatter には `source:` ブロック（`author` / `publisher` / `title` / `url` / `type`）を追加して出典を構造化（JS validator は unknown field を弾かないので非破壊で追加可能）。

## 「続けられない文体は致命的」

書き手が「これは自分の声ではない」と感じる文体は、技術が完成していても運用継続性が破綻する。**第 1 投稿は勢いで打てても、第 2 投稿で止まる**のが典型的な失敗パターン。

- ペルソナ → 文体ルール → 禁止表現 → テンプレ の順で**書き手が確信を持てる骨格**を定めた上で、AI に量産させる構造が継続運用に必要
- 量産の AI とペルソナ定義の AI を同じ skill 内で共存させる: PostCreation skill の SKILL.md に投稿パターン 5 種と良/悪の対比例を入れることで、AI が量産時にもペルソナ準拠を維持

## 横展開先

健康・サプリ系に限らず、書き手が**専門家ではないが詳しい領域**で発信するときに同じ構造が効く:

- 不動産・税務（書き手は素人 + 実体験者）
- 子育て・教育（書き手は親 + 専門家から学んだ）
- ガジェット・ツール紹介（書き手は使用者 + メーカー資料を読み込んだ）

「専門家のフリをせず、咀嚼して共有する立場」を明示する → 出典は権威付けではなく出発点として末尾脚注 → 命令クロージングを禁止 → 観察ポイントで締める、の 4 点が共通項。
