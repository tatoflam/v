# Google Sheets API の複数行ヘッダ（セル結合）対応

## Summary

Google Sheets API v4 の `values.get` は **セル結合を空セルで返す**。表計算で「上の行＝グループ見出し」「下の行＝詳細ラベル」として視覚的に結合されていても、API から見ると結合された 2 セル目以降は空文字になる。この挙動を知らずに最終行だけヘッダ化すると、グループ文脈が落ちて列名が `['納期', '', '納期', '']` のように潰れる。

## Details

### 実データで踏んだ具体例

`進捗管理表_master / 統合表` の 2 行ヘッダ（[[03_work/meguru-pm-report]] の PMMasterGrabber 実装で踏んだ）:

```
行1:  |案件情報       |設計フェーズ               |...|
行2:  |案件番号|建物名|設計開始|基本図承認|仮受予定|...|
```

API 返答は:

```
row1 = ["案件情報", "", "設計フェーズ", "", "", ...]  # 結合で2セル目以降が空
row2 = ["案件番号", "建物名", "設計開始", "基本図承認", "仮受予定", ...]
```

### 対応パターン

1. **上位行を前方埋め** (`ffill`) してグループ文脈を伝播
   - `["案件情報", "", ...]` → `["案件情報", "案件情報", "設計フェーズ", "設計フェーズ", "設計フェーズ", ...]`
2. **最終行を葉ラベル** として採用
3. **結合時は `/` 区切り** にして複合列名 `案件情報/案件番号` などを生成
4. ヘッダセル内の `\n`（狭幅コラムの cosmetic wrap、例 `設計\n開始`）は **自動除去**

### 設定で切り替え可能に

単一行ヘッダと複数行ヘッダが混在する（PMMasterGrabber では統合表のみ 2 行、他 2 シートは 1 行）場合、設定ファイルで
`header_row: int | list[int]` を受け付けて分岐。実装例は
`plugins/pm-master-grabber/scripts/infrastructure/sheets_client.py` の
`build_headers()`。

### 注意

- セル結合を使わずに上下行で同じテキストを反復入力しているシートは前方埋めすると想定外の結合が発生する。まず API 応答を生で見て、空セルがどこに出るか確認してから分岐ロジックを組む
- `\n` 除去は cosmetic であり、意味論的な改行（複数単語の列名）がある場合は誤削除になる。実ヘッダを眺めて判断
