# Threads 投稿 4 役割ローテと共感の解像度

## Summary

カピおじ (カピパラおじさん) が note で公開した「**トキメキ沼子。** アカウント分析」を要約し、account_1 (oheyamemo / interior、[[03_work/threadsposts]]) に転用可能な形に落とした学び。沼子。は楽天アフィリ Threads アカウント開設 6 日でフォロワー 3,000 を獲得した事例。要約レポートの実体は ThreadsPosts repo の `docs/resource/k02_05.md`。

## なぜ伸びたか — 構造的要因

| レイヤー | 仕掛け |
|---|---|
| プロフィール | 名前=キャッチー / アイコン=シンプル目立つ / 本文="フォローメリット × 共感 × ユーモア" の 3 点セット |
| 投稿ローテ | ①挨拶(共感) → ②商品紹介 → ③有益情報 → ④商品×3 → ⑦日常 → ⑩商品 でバズ |
| アルゴ最適化 | 外部リンクなし投稿 (③有益情報) で素地を作り、商品投稿のリーチを底上げ |

## Threads アルゴリズムの含意

- **一度見たアカウントの投稿は優先表示される** — 共感投稿で先にリーチを取ってから収益投稿を打つ順序が必須
- **外部リンク投稿はアルゴ的に不利** — 有益情報の URL はコメント欄に逃がす
- **「アカウントパワー」は蓄積する** — 10 投稿目で爆発したのは前段 9 投稿の信頼貯金

## 投稿 4 分類 (役割設計)

1. **フォロワー獲得**: 共感 × 視覚 (複数画像で滞在時間↑)
2. **収益化**: 緊急性ワード (「夏前にはない」)、共感投稿の直後に置く
3. **有益情報**: 外部リンクなし、コメント欄にリンク
4. **ファン化**: フォロワー層の嗜好に寄せた日常 (おしゃれ好き層→おしゃれネタ)

## 共感の解像度

- 「音楽好き」ではなく **「ブルーハーツ好き」** — 固有名詞で人物像を立てる
- account_1 の "うち" 文体 ([[05_learn/persona-driven-content-rules]] の置換版) も同様で、"うちのソファ" ではなく **"うちのカリモク 60"** レベルまで降ろすと共感が立つ

## 警告: 表面パクリは死ぬ

> 「節子、そこじゃない。表面をパクるんやない。中身をパクるんや」

- 流行ワード (例: FUDGE) の借用は継続不可能
- 真似るべきは **共感ポイントの選び方 / 言葉選び / 投稿構成の意図** であって、語彙ではない
