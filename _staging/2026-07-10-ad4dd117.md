## きっかけ / 質問
「`config/profiles.yaml` で指定している通知先を一部変更したい場合、どうやって firebase 側に反映するんだっけ？」

## 手順・Runbook（通知先の変更反映）
`config/profiles.yaml` は **Firestore ではなく関数コードと一緒にバンドルされる**設定。編集したら Functions を再デプロイするだけ。

```bash
# 1. config/profiles.yaml の email_to などを編集
# 2. （任意）ローカル検証
python -c "from todobot.config import load_profiles; print(load_profiles('config/profiles.yaml'))"
# 3. 再デプロイ
scripts/deploy.sh functions
```

仕組み・注意点:
- `firebase.json:14` の **predeploy フック**が `config/profiles.yaml` を `functions/config/` にコピーし、`functions/main.py:90-95` がそのバンドル済みファイルを読む。
- **Firestore 書き込みや `firebase functions:secrets:set` は不要**（シークレットは LINE / Anthropic のキーだけ）。
- `config/profiles.yaml` は `.gitignore` 済み → ローカルの実ファイルを直接編集してデプロイする。
- **起動時バリデーション**が走る：`HH:MM` 形式、IANA タイムゾーン、`email_to` は非空、複数スロットは昇順かつ **14分以上間隔**。ここで落ちると関数が起動しない。
- どのプロファイルを有効にするかは **`ACTIVE_PROFILES` 環境変数**で切替可能（通知そのものを止めたいだけなら宛先編集より env で対応できる）。
