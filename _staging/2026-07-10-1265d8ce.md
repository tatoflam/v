## (2) Firebase デプロイトークンの .env 化（add-firebase-deploy-token）

ユーザ要望「トークンを .env に取得しておいてそれを使うようにしたい」。`firebase login` の定期失効に左右されず日次パイプライン（launchd headless）からも自動デプロイできるようにする。

- 実装: `plugins/drive-publisher/scripts/publish.py`（symlink 先 `meguruit/Plugins-Meguru` の実体）に `.env` ローダ `_load_dotenv()` を追加し main 冒頭で実行。`FIREBASE_TOKEN` を `os.environ` に注入（**実 env 優先**）。`deploy_site.py` は既に `FIREBASE_TOKEN` があれば `--token` を渡す実装済みだった → 不足は「トークン生成・.env 保存・publish 時読込」の 3 点のみ。
- `plugins/drive-publisher/.env.example` を新規作成（取得手順ドキュメント）。`.env` 実体は root Plugins-Meguru の .gitignore でカバー、600 権限で保存（gitignored、値はここには記載しない=CI 秘密トークン）。
- トークン生成は `firebase login:ci`（ブラウザ認証必須、Bash 非対話環境では `Cannot run login:ci in non-interactive mode` で不可）→ ユーザ自身がターミナルで実行して取得。
- 疎通確認: トークンセット状態で `python -m publish --run-date 2026-07-10` 再実行 → `deploy_site` が `--token` 経由でデプロイ成功。
- OpenSpec: `add-firebase-deploy-token` を起票（proposal/design/tasks/spec delta、capability=`weekly-report-distribution`）→ `openspec validate --strict` = valid → `openspec archive` で spec 確定反映。
- コミット（両リポ main、その後 push 済）:
  - Plugins-Meguru `1f39ccc feat(drive-publisher): .env 由来 FIREBASE_TOKEN で非対話 Firebase デプロイ`
  - MeguruPMReport `9126df0`（週次レポート成果物）/ `3fe3cc1`（openspec proposal）/ `816dbb1`（openspec archive）
- 留保: CI トークン方式は Google が段階的に非推奨化中。ユーザ判断で「とりあえずこのまま様子見」。頻繁に失効するならサービスアカウント（`GOOGLE_APPLICATION_CREDENTIALS`）移行 change を起票する方針を design.md/tasks に記載。

## 決定・学び
- 決定 2026-07-11: Firebase 非対話デプロイは `.env` 由来 `FIREBASE_TOKEN`（CI トークン方式）で実装。サービスアカウント移行は様子見。
- 学び: `firebase login:ci` は Claude の Bash（非対話）から実行不可 — ユーザのターミナル実行が必須。トークンは 600 権限・gitignored で保管。
- この作業でメモリ `firebase_token_env.md` を更新済み（publish の Firebase デプロイは drive-publisher/.env の FIREBASE_TOKEN を使う、失効時は firebase login:ci で再生成）。
