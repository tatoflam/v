# /weekly-report（2026-07-16）が権限制約で安全中断 — data/ symlink が許可ディレクトリ外に解決

## セッション概要（2026-07-16 02:24–02:27 UTC、約 3.5 分で中断）

`/weekly-report run_date=2026-07-16 diff_base=reports/2026-07-08_weekly_update.md` を実行したが、セッションの権限制約でデータ層に一切アクセスできず、レポート生成前に安全中断。ファイルは何も書き換えていない（state/reports は無傷）。失敗セッションだが、権限モデルの落とし穴の記録として価値がある。

## 何がブロックされたか（根本原因）

1. **`data/pm_master/` / `data/jooto/` 読み取り不能** — consolidate-grabber-plugins 統合後、リポジトリの `data/` は `~/meguru-data`（`MEGURU_DATA_ROOT`）への symlink。このセッションの許可ディレクトリは repo のみだったため、symlink の解決先が許可外となり `ls` / `cat` / Read すべて auto-deny。milestone.json / integration.json / overdue.json / progress_checklist.json が読めず、工程軸グリッド・条例マトリクス・Milestone 連動サブ行・負荷ビューが構築不能。
2. **`python3` 実行が承認待ちで拒否** — repo 内の `scripts/weekly_anchor.py` ですら実行不可。baseline sync（drive-publisher venv の `python -m sync_baseline`）と carry-forward 差分モジュールが起動できず、コマンド仕様（baseline sync 失敗時はユーザー確認）に従い中断。

## 読めた範囲での状態確認（正常だった事実）

- repo 内 `state/` / `reports/` / `config/` は読み取り可能
- `state/weekly_anchor.json` のアンカー = `2026-07-08` — 指定 `diff_base` と一致し引数は妥当
- 前回 publish は 2026-07-15（`published_doc_id` 登録済み）

## 決定事項・判断

- **劣化レポートを出さない判断**: Gmail MCP + repo 内ファイルだけで工程軸なし・Milestone なしのレポートを出すことは可能だが、`state/latest_summary.md` / `state/latest_state.json` を劣化内容で上書きすると翌日以降の日次アンカー差分基盤を壊すため、明示指示なしには実行しない — 安全側で中断を選択。
- **再開手順**: `claude --add-dir /Users/tato/meguru-data --add-dir /Users/tato/repo/github/meguruit/Plugins-Meguru` で再起動して再実行（推奨）、または対話モードで都度許可。実際に直後のセッション（42a1be27）で再実行し成功。

## 教訓

- 既存知見「headless claude -p は 3 点セット必須（--add-dir×2 / bypassPermissions / prompt は stdin）」の裏付け事例。**--add-dir なしで /weekly-report を起動すると data/ symlink 越しのアクセスが全滅して必ず中断する**。symlink 先（`~/meguru-data`）は cwd 許可では届かない。
- 中断時の安全性は担保: /weekly-report は事前確認段階で失敗を検出し、state/reports を汚さずに止まる設計が実際に機能した。
