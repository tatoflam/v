## 何を作るシステムか（MeguruInvoice）

取引先請求書（Drive上のスキャンPDF、手書き含む）を一括読み込みし、レビューを経て Google スプレッドシートへ取り込む社内向け Web システム。ユーザーは GWS 組織アカウント限定。

処理フロー: ①共通フォーマットの請求書テンプレを取引先へ配布 → ②取引先が請求書を送付（メール/印刷） → ③スキャンPDFを Google Drive の組織共有フォルダに月別配置 → ④一括読込 → ⑤AI抽出（手書き含む、信頼度付き） → ⑥Firestore保存 → ⑦レビュー・二重読込チェック・明細採用選択 → ⑧確定後スプレッドシート書込 → ⑨読込済みは完了フォルダへ移動し次回対象外に。

## 主要な決定事項

- **DB廃止・Firebase化（design D5）**: Cloud SQL を廃止し、**Firebase App Hosting（GitHub連携の自動デプロイ、ゼロスケール）+ Firestore（完全従量課金、本件規模なら実質無料枠内）** に変更。理由: 常時稼働DBは運用コストが高い。「スプレッドシートをDB代替に」案は、一意制約とトランザクションがなく二重取込防止という中核要件を安全に実装できないため**不採用**（design D5 に記録）。**Cloud SQL 再導入は禁止**。
- **AI抽出エンジン（design D4, tasks グループ1）**: 公開ベンチマーク調査の結果、請求書項目抽出は Gemini 94%（GPT+OCR構成 91%）で優位、純粋な手書き認識は GPT-5 95%（Gemini 93%）でやや優位と拮抗。日本語手書き×自社テンプレートでの優劣は公開情報からは断定不可。方針: **第一候補 Gemini**（GWS/GCPとの契約・認証一元化、PDF直接入力）、ただし**要件定義時に実サンプル各10件で Gemini vs OpenAI を PoC ベンチマークして最終確定**、エンジンは差し替え可能に抽象化。
- **見積前提**: 1人月=160時間、1時間=10,000円。**合計664時間（約4.2人月）= 初期構築費 664万円（税別）**。保守は軽保守（月16h）16万円/月、標準（月40h）40万円/月。（初版は676h/425万円想定 → Firebase化で基盤・認証を圧縮しつつ AI PoC 24h 追加して664hに改定。）
- **インフラ運用費**: 月約2,000〜4,000円（旧 Cloud SQL 構成の1/4以下）。
- **共通請求書フォーマットの準備**を前提事項の筆頭に追加。テンプレに請求番号欄・物件コード欄を設けることを推奨（重複判定精度に影響、確認事項）。
- **完了フォルダ移動の順序制約**: シート書込成功後にのみ完了フォルダへ移動（handoff 不変条件）。
- リポジトリは **private**（社内システムのため）。
