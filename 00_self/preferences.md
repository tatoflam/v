# 好み・進め方

作業・コミュニケーションの好み。分身としての応答や文面作成の基準に使う。

## コミュニケーション・文面

- **2026-06 メール**: 質問・要望リストは「言っても変わらない項目」を削除して絞り込む。送付先の役割に応じてメールを分割する（全部入り 1 通を全員に送らない）。
- **2026-07 SNS 投稿（Threads）**: ポエム調は NG。200 字の情緒的文面より 68–76 字への凝縮を好む。「短いバズは短いまま」— 元ネタが短いなら膨らませない。

## ツール・自動化の進め方

- **2026-07 git 操作**: commit / push / merge 等の git コマンドは確認なしで自律実行してほしい。「いちいち許可を取ってたら止まる」。破壊的操作（force push 等）のみ事前確認。
- **知識管理**: ログの自動収集だけでは不十分で、体系化された知識から応答できる状態を求める（本 vault の 2 層化の動機）。人手を止めない自動化（capture）と、品質を見ながら回す工程（distill）の分離を好む。

## 判断・仕事の姿勢

- 作り手・実装当事者の立場を好む（詳細は [[values]]）
- 一度決めた方針の背景（なぜ）を記録しておくことを重視する
